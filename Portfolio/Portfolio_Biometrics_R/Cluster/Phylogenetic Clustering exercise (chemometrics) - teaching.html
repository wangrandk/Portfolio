Analysis of the Simple Data Set

In this section, we will redo the analysis we performed in the handout analysis, to see that the software gets the same result as you did.

Step 1: Install and load required packages

First: Start R (in terminal window, GUI, or RStudio)

Then, install and load the packages ape and phangorn, which contain functions for phylogenetic analysis: 

install.packages("ape")
install.packages("phangorn")
library(ape)
library(phangorn)

Step 2: Download sequence data

The file linked below contains the same alignment you analysed manually in the handout exercise. Save the content of file to a file named "simple.txt" in the R working directory (for instance copy and paste from the webpage below to a text editor, and save in the proper location):

simple.txt

Step 3: Load sequence alignment into R

Read the alignment into a variable named "simpledat": 

simpledat = read.phyDat("simple.txt", type="DNA", format="fasta")

Step 4: Compute distance matrix from alignment

The function dist.hamming() computes a distance matrix from an alignment. 

dm = dist.hamming(simpledat, ratio=FALSE)
print(dm)

The option ratio=FALSE means that distances should be counted as the absolute number of changes instead of as the fraction of changed sites (i.e., the number of changes divided by length of alignment). Is this the same matrix you got manually?

Step 5: Compute phylogenetic tree from distance matrix

Use neighbour joining to compute a tree from the distance matrix, and plot the tree in unrooted form. Is this the same tree you got in the handout exercise? 

simpletree = NJ(dm)
plot(simpletree, type="unrooted", rotate.tree=90, lab4ut="axial")

The option lab4ut="axial" plots tip names in the direction of the branches

Step 6: Compare branch lengths to manual result

Branch lengths are stored as an attribute of the tree. Specifically they are available as a vector of lengths for the consecutively numbered branches in the attribute treename$edge.length: 

print(simpletree$edge.length)

By converting this numeric vector to a character vector, and using the function edgelabels() we can plot these branch lengths on the tree. 

brlens = as.character(simpletree$edge.length)
edgelabels(brlens, bg="white", frame="circle")

Are these the same values that you got in the manual exercise?

The Phylogeny of HIV

In this exercise you will analyze the evolutionary relationship between HIV-related viruses from man and monkeys:

Acquired Immune Deficiency Syndrome (AIDS) is caused by two divergent viruses, Human Immunodeficiency Virus one (HIV-1) and Human Immunodeficiency Virus two (HIV-2). HIV-1 is responsible for the global pandemic, while HIV-2 has, until recently, been restricted to West Africa and appears to be less virulent in its effects. Viruses related to HIV have been found in many species of non-human primates (monkeys, apes, ...) and have been named Simian Immunodeficiency Virus, SIV. HTLV-1 is another, more distantly related, member of the family of retroviruses to which HIV and SIV belong.

The "Pol" gene, which is present in the genome of all these viruses, encodes three different polypeptides important for the viral life cycles: integrase, reverse transcriptase, and protease.

In this exercise you will use a data set consisting of 21 different POL-polyprotein sequences from HIV1, HIV2, chimpanzee SIV, sooty mangabey SIV, and HTLV-1. It is available via this link:

pol21.fasta

Step 1: Align protein sequences

Align the Pol sequences using the mafft server at EBI with default settings (copy/paste the alignment into the input box on the mafft server page, and click submit) . Let Output format be "Pearson/FASTA".

Once the alignment is done, save the resulting alignment as a fasta file: right-click the "Download alignment file" button on the mafft output page, and then save the file using "Save linked file as" (or whatever it is called in your particular browser) under the file name "pol21_aligned.txt".

Make sure that:

  • The alignment is saved in TEXT format
  • The file is located in your R working directory
  • The file is named pol21_aligned.txt

Step 2: Read alignment into R

In the window where you are running R: read the aligned data into the variable pol21: 

pol21 = read.phyDat("pol21_aligned.txt", type="AA", format="fasta")

In case of trouble:

  • Is the file in your current working directory?
  • Is the file in TEXT format? (Not HTML or similar)?
  • Is the file named pol21_aligned.txt?

Step 3: Compute distance matrix from alignment

Compute a distance matrix from the protein sequence alignment: 

dm = dist.hamming(pol21, exclude="pairwise")

The option exclude="pairwise" affects how gaps are treated: We do not want to count each gap character as a mutation. This option ensures that for each pair of sequences we compare gap positions are ignored (or excluded).

Step 4: Compute tree from distance matrix

Compute neighbor joining tree and plot as unrooted tree: 

hivtree = NJ(dm)
plot(hivtree, type="unrooted", lab4ut="axial", cex=0.8)

Step 5: Root tree using outgroup

Use HTLV as outgroup and plot tree in rooted form: 

hivtreerooted = root(hivtree, "HTLV")
plot(hivtreerooted, type="phylogram", cex=0.8)

The rationale for using an outgroup to place the root of the tree is as follows: our data set consists of sequences from HIV-1, HIV-2, SIV and HTLV. We know from other evidence that the lineage leading to HTLV branched off before any of the remaining viruses diverged from each other. The root of the tree connecting the organisms investigated here, must therefore be located between the HTLV sequence (the "outgroup") and the rest (the "ingroup"). This way of finding a root is called "outgroup rooting. (As another example consider building a tree from mammal sequences - in that case a bird would be a suitable outgroup).

Step 6: Add labels to two major clusters

Label internal nodes, so we know what node numbers we want to use when labelling the two main clades: 

nodelabels(cex=0.8)

Now, re-plot the tree and label the HIV1 and HIV2 clusters (using the nodenumbers we saw above): 

plot(hivtreerooted, type="phylogram", cex=0.8)
nodelabels(c("HIV-1", "HIV-2"), c(23,32), bg="white", cex=0.8)

Step 7: Consider evolutionary origin of HIV-1 and HIV-2

The original assumption was that HIV-1 and HIV-2 were closely related variant viruses. Your tree does not support this hypothesis, but what does it then indicate? When you've thought about it, check this link for an explanation: Origin of HIV-1 and HIV-2

Step 8: Optional: Save tree and inspect tree in external viewer

It is also possible to save the tree to a Newick formatted file, and then view it in external programs. If you'd like to try this, first install the program FigTree (use the version for your operating system).

Then, from within R, write the tree to a file: 

write.tree(hivtreerooted, file="hivtree_root.txt")

You can now start FigTree and open the saved file. FigTree has options for modifying fonts, line widths, etc, and also for manipulating the tree (re-rooting, displaying clusters in "cartoon" format, etc).

Optional extra exercise: build your own tree

If you have time to spare and want to experiment with building your own tree from scratch try this task: Create a rooted phylogenetic tree with the dataset linked below, which consists of DNA sequences encoding the ribosomal protein L18 from a number of different species. L18 forms part of the 60S subunit of the ribosome. (The sequences used here are not the complete coding sequence, but lack the first 90 nucleotides or so).

  • L18_CDS.fasta

Recall that the steps needed to get a tree are:

  • Aligning the sequences
  • Computing a distance matrix from the alignment
  • Building a neighbour joining tree from the distance matrix
  • Optionally: Rooting the tree on an outgrip
  • Plotting the tree
